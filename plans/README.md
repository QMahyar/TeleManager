## Dependency notes

- **002 requires 001** because the tests assert the corrected middleware
  exemption list, setup-requires-current-password contract, and server-side
  session invalidation on logout. Writing tests against the pre-001 behaviour
  would lock in the bugs.
- **003, 004, 005, 006, 007** are independent of each other and of 001/002.
  Prefer 001→002 first (security), then 003→004 (correctness/data min), then
  005–007.
- **008 must land before 009, 010, and 011** — it defines the tokens
  (`--warning`, `--info`, `--primary-text`, `--sunset-ink`, strengthened
  `--ring`) and their Tailwind `@theme` mappings that those plans' classes
  resolve against. Running 009–011 first produces classes that silently
  resolve to nothing.
- **009, 010, 011 are independent of each other** but 009 and 010 touch two
  of the same files (`status-bar.tsx`, `run-banner.tsx`, `ui.tsx`,
  `overview-screen.tsx`, `action-picker.tsx`) on different class fragments;
  execute them sequentially (009 → 010 → 011), not in parallel worktrees, to
  avoid merge friction. Each plan says which fragments belong to the other.
- **012 is fully independent** and can run at any point.

## Findings considered and rejected

- Localhost API without multi-user auth: **by design** (AGENTS.md + TrustedHostMiddleware).
- `send_media` accepting absolute host paths: **by design** (local single-operator tool; comment at `telegram_actions.py`).
- Cookie `secure=False`: **by design** (HTTP localhost, no TLS).
- Splitting `telegram_actions.py` / `schedules_service.py` solely for size: **not worth it** until Phase 2 forces a natural seam.
- Untracked root `probe_*.py`: operator scratch, not product debt.

From the 2026-07-14 design audit (Hallmark), found but NOT planned:

- Confirmation dialogs before queue/delete actions flagged by the taste
  catalog: **by design** — AGENTS.md's guarded-actions rule requires
  confirmation before bulk Telegram actions; domain rules beat taste rules.
- Pure-white `--card` on the warm canvas: **by design** — Arc's documented
  surface language ("elevated rounded white cards", AGENTS.md).
- Success toasts after visible effects (e.g. "Account renamed."):
  **deferred, product call** — an operator console arguably wants explicit
  ack for destructive/remote effects; revisit only if toast fatigue is
  actually reported.
- `theme-provider.tsx` hex literals (favicon tile + canvas fallback):
  **correct as-is** — favicon renders in browser chrome detached from CSS
  custom properties (file comment explains).
- Light-mode `--primary-foreground` white-on-coral in the segmented control
  (`settings-screen.tsx:329`): **deferred** — fixing it means rethinking the
  `--primary-foreground` pairing across sidebar/badges; tracked in plans
  009/011 maintenance notes.
- Spinners (`animate-spin`) under `prefers-reduced-motion`: **kept** —
  loading indication is information, not decoration.
